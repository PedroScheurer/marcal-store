# gateway-serviceO gateway-service é a camada de borda (API Gateway) unificada do ecossistema de microsserviços, desenvolvida em Java 25 e alimentada por Spring Boot 4.0.6. Utilizando o Spring Cloud Gateway (WebFlux), ele atua como o ponto único de entrada para todas as requisições de clientes externos, sendo responsável pelo roteamento dinâmico inteligente, balanceamento de carga reativo e validação centralizada de segurança através de tokens JWT.## 🛠️ Tecnologias e Dependências PrincipaisJava 25 & Spring Boot 4.0.6: Base reativa de altíssima performance para o ciclo de vida da aplicação.Spring Cloud Gateway Server (WebFlux): Mecanismo assíncrono e não-bloqueante para roteamento de fluxos baseado em predicados e filtros.JSON Web Tokens (JJWT 0.13.0): Componente integrado para decodificação, checagem de integridade e assinatura criptográfica de claims (jjwt-api, jjwt-impl, jjwt-jackson).Spring Cloud Netflix Eureka Client: Integração ao servidor de Service Discovery para permitir a resolução de rotas via nomes lógicos de serviços com balanceamento de carga (lb://).Spring Boot Actuator: Exposição de métricas internas, status das rotas e monitoramento de saúde em tempo real.## 📂 Estrutura de Diretórios do ProjetoBaseado na organização estrutural visível na árvore do projeto (arquivos image_43d2db.png e image_43d2e2.png), o microsserviço divide suas responsabilidades da seguinte forma:Plaintextgateway-service/

## 🛣️ Malha de Roteamento DinâmicoO gateway intercepta as requisições na porta 8765 e despacha-as dinamicamente para as instâncias ativas no Netflix Eureka utilizando o prefixo balanceado lb://.Abaixo estão as regras explícitas de encaminhamento configuradas na classe GatewayConfig:Caminho da Requisição (Path)Microsserviço Destino (URI)Descrição do Fluxo/auth/**lb://auth-serviceFluxos de autenticação, login e cadastro./products/**lb://product-serviceGerenciamento e listagem de produtos./ws/products/**lb://product-serviceWebSockets / Comunicação estendida de produtos./currency/**lb://currency-serviceConversão de moedas e consultas ao BCB./ws/currency/**lb://currency-serviceCanal reativo / WebSocket de moedas./ws/orders/**lb://order-serviceProcessamento e eventos de pedidos/ordens.## 🛡️ Segurança Centralizada (JwtUtil)O Gateway impede que requisições maliciosas ou sem autenticação cheguem aos microsserviços internos. Através da classe utilitária JwtUtil, o gateway descriptografa e valida a assinatura criptográfica dos tokens utilizando o algoritmo HMAC-SHA com uma chave privada configurada localmente.O processo lança exceções de infraestrutura customizadas quando anomalias são encontradas:TokenExpiredException: Disparada automaticamente quando o tempo de vida do token expirou (ExpiredJwtException).InvalidTokenException: Disparada quando o payload foi violado, a assinatura é incompatível ou o formato é malformado (JwtException).## 📊 Monitoramento Operacional e TelemetriaO ecossistema expõe dados para ferramentas de monitoramento e auditoria na raiz do barramento. As propriedades do arquivo application.yml configuram a exposição dos seguintes endpoints do Actuator:Saúde Completa da Malha (Probes Liveness/Readiness): GET http://localhost:8765/actuator/healthStatus dos Circuit Breakers: GET http://localhost:8765/actuator/circuit-breakersMétricas Gerais de Tráfego: GET http://localhost:8765/actuator/metrics## ⚙️ Requisitos para Inicialização LocalCertifique-se de utilizar o Java 25.Garanta que o servidor de registro Eureka Server (discovery-service) esteja ativo na porta 8761 antes de subir este serviço, permitindo a correta amarração das rotas dinâmicas.Suba a aplicação utilizando o wrapper do Maven:Bash./mvnw spring-boot:run
